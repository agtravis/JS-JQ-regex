Linking jQuery & Document Ready

Linking the jQuery library

jQuery code won't work on its own. jQuery is essentially a shorthand version of javaScript, but the browser doesn't inherently recognize it without linking the page to the library. When javaScript is used, it can either be inbetween <script> tags, or if extensive, a *.js file can be linked as in <script src="js/scripts.js"></script>. Similarly, this is how the jQuery library can be linked.

The library can be linked in one of two ways. First, to a *.js page hosted by the webpage, or locally accessible. The benefit of this is that the library can be trimmed down (with some expertise) to be narrowed to just what is required for functionality. The library is huge, and loading all of it unecessarily can be a drag on the browser.

Alternatively, there are many libraries hosted on line available for webpages to publically link to. the link would look like this: <script src="https://ajax.googleapis.com/ajax/libs/jquery/3.1.0/jquery.min.js"></script>. I am using a more up to date library hosted by Google. However other libraries are available, either older versions, or perhaps more up to date. Other places instead of Google could be used to host as well, such as Microsoft. The most important detail is the version.

It is recently conventional to place this link at the bottom of the HTML, right above the closing </body> tag.

Document Ready

Using this code keeps the whole of your jQuery code to one side until the whole document has loaded in the browser. Since the jQuery code has been designed to work on a fully formed webpage, you want that webpage and the associated DOM to be ready to go before.

In order to do that, you contain all of your jQuery code within the following annonymous function:

$(document).ready(function() {
 jQuery code goes here...
});

ALL of my jQuery is contained within this function, all 1500 lines of it.
