This uses HTML <form> tags to first take user input in a field and store it as innerHTML content, then uses that stored content to change the innerHTML content of a paragraph, and then also auto-populate another field.

The autocomplete has been disabled, and the field is coded to change color while in use, and change color again when not in use. This, for demonstration purposes, is undertaken using in-line javaScript, frowned upon by convention but illustrated here.

Type anything and click Submit below:

This will change to user input....

It is worth noting that even though the result would be the same, the code is generating the paragraph based on the first field input, and then the latter field input based on the paragraph content. The alternative would be to get both the contents from the initial input. This means the code MUST be in the correct order, otherwise the paragraph content will not have been generated before the secondary field is attempting to get it.
