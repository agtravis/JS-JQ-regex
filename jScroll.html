Monitor scroll position & trigger scroll event

jQuery methods used in this code:

  • .animate({})
  • .css()
  • .scroll()
  • .scrollTop()

The scroll position is
0
pixels from the top.

This code does two things. First, it tells the browser that when the page is scrolled, even by one pixel, that something will happen. This would be the event listener '.scroll()'.

Next, it tells the page what to do on the scroll. In this case, the scroll position (using '.scrollTop()') is being displayed in a paragraph (the paragraph has been coded in CSS to remain in the same place), and this will change as the page scrolls.

When the page scrolls down past a certain point ('.scrollTop(200)'), an image (within a div) is coded to fly up from the bottom. When the page scrolls back up to that point or above, the image will fly back off screen. NOTE: Due to the fact that the event listener is 'scroll', this will cause a few issues. A scroll event means that with every pixel the scrollbar moves, code is run to check for criteria. This can be problematic with a lot of code or data. Also, in the animation, we need to '.stop(true)' the initial animation to stop the animation UP and be able to replace it with an animation DOWN, and the 'true' part means all animations are cancelled.

I have also coded a jQuery scroll function for every single page on this site. When the page is viewed at full size, the header will stay at the top all the time regardless of scroll. When the window is too narrow (defined using @media in CSS), the header will scroll up, and when it is fully off screen, the mini navigation bar will slide in to view. When the user scrolls back up and the header comes in to view again, the navigation bar disappears.
